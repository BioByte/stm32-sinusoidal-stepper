# stm32f407-baremetal-template

![ARM](https://img.shields.io/badge/ARM-Cortex--M4-blue)
![Build](https://img.shields.io/badge/build-make-green)
![License](https://img.shields.io/badge/license-MIT-yellow)
![IDE](https://img.shields.io/badge/IDE-VSCode-blue)

Bare metal STM32F407 development environment using ARM GCC, Makefile, CMSIS, VSCode and J-Link without vendor IDEs.

This repository demonstrates how to manually configure a complete embedded firmware workflow, including:
- ARM GNU Toolchain
- CMSIS Core + Device
- Startup Assembly
- Linker Script
- Makefile build system
- Flash and debug with J-Link + GDB
- VSCode integration
- Build optimization with ccache and LTO

The purpose of this project is to demonstrate the real firmware flow behind embedded systems development instead of relying on autogenerated IDE configurations.

# Why Bare Metal?

This project focuses on understanding the real firmware execution flow,
toolchain interaction and hardware abstraction layers instead of relying
on autogenerated vendor IDE projects.

# Included Components

## CMSIS

Provides:

* Cortex-M4 core definitions
* Compiler intrinsics
* Peripheral register mapping

## Startup Assembly

Responsible for:

* Vector table
* Stack initialization
* Reset_Handler
* SystemInit() call
* main() call

## Linker Script

Defines:

* FLASH memory layout
* RAM memory layout
* .text / .data / .bss sections
* Stack position

## Makefile

Handles:

* Compilation
* Linkage
* Binary generation
* Flashing
* Debug support
* Memory analysis
* Build optimization
